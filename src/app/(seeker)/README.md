# Seeker Surface

Seeker-facing pages include discovery, directory, map, saved, profile, chat, the local-first plan workspace, and the gated execution dashboard.

This surface must keep crisis routing and safety messaging prominent.

Execution-layer rules for seeker surfaces:

- Plan items may link to stored ORAN service records, but may not invent provider facts.
- Manual plan items are explicitly user-authored.
- The first execution slice is local-first on the device.
- Reminder scheduling and dashboard summaries are local-first in the current slice.
- Eligibility caution still applies anywhere a linked service appears inside a plan.
