# Governance

This folder defines **how ORAN is operated** (process, roles, definition-of-done). These docs are part of SSOT as “Operational workflow.”

## Key docs

- `GOVERNANCE.md` — working agreements and guardrails
- `OPERATING_MODEL.md` — how changes are proposed, implemented, and verified
- `ROLES_PERMISSIONS.md` — role boundaries (Seeker/Host/Community Admin/ORAN Admin)
- `PAGE_DEFINITION_OF_DONE.md` — acceptance criteria for UI pages
- `TAGGING_GUIDE.md` — tagging taxonomy used across the system
- `PROJECT_BOARD.md` — project/board conventions
- `COMMIT_SERIES_PLAN.md` — (if used) how commit series are structured
